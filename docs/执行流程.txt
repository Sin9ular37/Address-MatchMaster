准备数据：从 Excel/CSV 读取高德 POI 表与快递原始地址表，字段名称通过 config.yaml 中的映射配置（省/市/区/街道/门牌、POI 坐标等）统一。
地址标准化：AddressNormalizer 使用规则+结巴分词+拼音将地址清洗成统一文本、Token 和门牌信息，POI 数据同样会生成标准化字段。
倒排索引召回：对所有 POI 构建一个简单倒排索引，按 Token 命中数取 Top N 候选（如需，可在配置里启用向量召回扩展，但当前默认只是倒排）。
候选打分：CandidateScorer 根据 Token 覆盖率、编辑距离、门牌一致性等特征对候选 POI 评分；得分最高且超过阈值的 POI 即认为匹配成功，否则留空以便人工复核。
结果导出：整个流程由 geo-matcher config.yaml 驱动，批量处理完 40 万地址后，将匹配结果（包含匹配状态、得分、候选列表等）写入配置指定的输出文件，并可结合 chunk_size、workers 做分批/并行处理。
总体上流程就是：配置 → 数据加载 → 地址清洗 → 候选检索 → 特征打分 → 导出匹配结果。
